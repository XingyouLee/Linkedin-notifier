# PM Review: LinkedIn Notifier — 2026-04-27

## 1. Current Product Assessment

### What does this tool actually do for users?

The LinkedIn Notifier is a personal job-discovery engine. The end-to-end flow:

1. **Scanning**: On a schedule, scrapes LinkedIn guest search results for configured search terms (e.g., "Data Engineer", "ETL/ELT Engineer") within the Netherlands.
2. **Job Description Fetching**: For each discovered job, fetches the full job description from LinkedIn's jobPosting API.
3. **LLM Fit Scoring**: For each profile-job pair, sends the job title, description, resume, and a detailed fit prompt to an LLM. The prompt enforces conservative scoring with explicit rules for seniority, experience years, language blockers, and false-positive reduction.
4. **Conservative Post-Processing**: Python-side caps (`_apply_fit_caps()`) enforce additional policy — if the LLM scored a job too high despite a seniority mismatch or language blocker, the code overrides it downward.
5. **Discord Notification**: Jobs scoring "Strong Fit" or "Moderate Fit" are sent as Discord messages to a configured channel.
6. **Web Dashboard (Django portal)**: A read-only view of notification runs, job match details, score analytics, and a per-job pipeline explorer.

### What's working well?

- **Fully automated pipeline**: From scan to notification, zero manual steps.
- **Sophisticated fit scoring**: The LLM prompt is unusually detailed — 400+ lines of instructions for experience extraction, seniority mapping, language detection, and false-positive auditing. This is the core IP of the product.
- **Multi-profile architecture**: Clean separation of canonical job data from per-profile state (jobs vs profile_jobs).
- **Conservative by design**: The `_apply_fit_caps()` post-processing and language blocker enforcement bias toward false negatives over false positives.
- **Test mode infrastructure**: `LINKEDIN_TEST_MODE` with bounded caps and test-only profiles allows end-to-end smoke tests.
- **Durable notification runs**: The `notification_runs` / `notification_run_jobs` table pair provides audit trail.

### Current UX assessment

**Discord notifications**: Functional but noisy. No throttling, no digest mode, no dismissal/archive mechanic, no "not interested" to prevent re-notification.

**Django Dashboard**: Cleanly designed (HTMX-based lazy loading) but feels like a debug/administration tool rather than a daily user product. No concept of "my jobs that I should apply to" — the dashboard shows historical runs, not a personal queue.

## 2. Product Gaps and Pain Points

### Critical gaps

1. **No application tracking**: The product tells you about good jobs but provides zero support for the application process. Did you apply? What was the outcome?

2. **No dismissal or "seen" mechanic**: Every qualifying job is notified forever. No way to mark a job as "not interested", "already applied", or "dismissed".

3. **No digest or throttling**: Every qualifying job is sent as a separate Discord message. No daily/weekly digest, no bundling, no quiet-hours mode.

4. **Discord-only notification**: If you don't use Discord actively, you miss everything. No email fallback, no push notification.

5. **No dashboard as daily driver**: The dashboard serves no daily user need. No "jobs today" feed, no "recommended for you" list, no bookmark/save functionality.

### Experience breakdowns

- **Scanning reliability**: Public LinkedIn scraping is fragile. No alerting for failed scans.
- **Cold start latency**: When a new search config is added, no preview of what the scan will find.
- **Fit latency**: LLM fitting runs with some concurrency but no user visibility into "how many jobs are still being processed."
- **Duplicate notifications**: If a job stays in search results across multiple scan runs, it could be re-notified.
- **No feedback loop**: User cannot tell the system "this was a good/bad recommendation."

## 3. Feature Opportunities

### Email notifications — Effort: S | Impact: High
Send HTML email digest or individual job alerts. Build as a pluggable notifier backend.

### Web dashboard improvements

- **Pipeline status bar** (S | Medium): Surface last scan time, jobs in queue, fitting progress, Discord delivery status prominently.
- **Job analytics and trend charts** (M | Medium): Fit score trends, company distribution, search term effectiveness, notification delivery reliability.
- **Personal job feed** (M | High): Replace run-centric dashboard with job-centric feed: qualified jobs sorted by recency/score, with inline actions (dismiss, bookmark, mark applied).

### Self-service profile management — Effort: M | Impact: High
Expand settings to support: add/remove/edit search configs, geo-location picker, resume upload, fit prompt version history, per-profile notification channel config.

### Multi-channel notifications — Effort: M | Medium
Abstract notification backend: Discord (current), Email (SMTP), Slack webhook, Telegram bot, Mobile push (ntfy).

### Application tracking — Effort: M | High
Add `applications` table: profile_id, job_id, status (applied/screening/interview/offer/rejected), applied_at, notes. Close the loop from discovery to outcome.

### Better search filtering — Effort: S | High
Add: salary filter, company type filter, posted date range, employment type (full-time, contract, internship).

### Saved/bookmarked jobs — Effort: S | Medium
Persist a `saved` boolean on `profile_jobs`. Saved jobs appear in a dedicated dashboard view.

### Weekly labor market report — Effort: L | Medium
Generate a weekly email digest: companies hiring in your space, most common required skills, salary signals, search term effectiveness trends.

## 4. Prioritized Roadmap

### Quick wins (low effort, high impact) — Top 3

1. **Job bookmark/save (S | High)**: Single highest-leverage UX improvement — gives user a way to act on a notification.
2. **Pipeline status bar on dashboard (S | Medium)**: Eliminates "is the pipeline working?" uncertainty.
3. **Email notification channel (S | High)**: Natural notification channel for job search. Reuse existing notification run infrastructure.

### Strategic investments (medium effort, high impact) — Top 3

1. **Personal job feed on dashboard (M | High)**: Transforms portal from monitoring tool into daily "what should I apply to" interface.
2. **Dismissal and feedback loop (M | High)**: Prevents notification fatigue and makes the system learn from user behavior.
3. **Application tracking system (M | High)**: Closes the loop from job discovery to application outcome.

### Moonshots (high effort, transformative) — Top 2

1. **Weekly labor market report (L | Medium)**: Transforms tool from passive notifier to active market informant.
2. **Smart notification routing and prioritization (L | High)**: Score-based routing, time-based routing, deduplication, pluggable routing layer.

### What should NOT be built

- LinkedIn login integration (risk account suspension)
- Interview scheduling integration (premature)
- Full multi-tenant SaaS (drowns core value)
- Auto-apply / one-click apply (ethically questionable)
- Custom resume builder (separate Resume Matcher project handles this)
- LinkedIn messaging / InMail automation (aggressive bot behavior)
- Salary negotiation assistant (out of scope)

## 5. Competitive Context

### vs. LinkedIn native job alerts

LinkedIn job alerts: free, reliable, keyword-based matching, no personalization, no scoring.

LinkedIn Notifier: deep personalized LLM-based scoring against YOUR resume and target roles, conservative filtering, full audit trail, privacy (your resume stays local), company/title-level blacklisting, multi-profile.

**Core differentiation**: The LLM-based fit scoring system. LinkedIn sends every "Data Engineer" posting. The Notifier reads the full JD, evaluates it against your experience level, seniority target, skills, language abilities, and only alerts when the match is genuinely plausible.

**Secondary value**: Total control — search terms, location, company blacklist, title excludes, fit prompt, notification channel.

**Weaknesses**: Scan reliability (HTML changes break it), job coverage (guest API may not return all results), no social proof, no easy apply, notification latency (minutes-to-hours).

This tool is a **complement** to LinkedIn alerts, not a replacement. LinkedIn casts a wide net; the Notifier is the precision filter.
